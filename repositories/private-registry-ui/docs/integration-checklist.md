# Integration checklist

- [ ] Real Entra login returns to `/login/oauth2/code/entra` and creates a server-side session.
- [ ] Session, counts, search, details, documentation, governance, and SSE are filtered by the selected APM.
- [ ] Users without entitlements receive an authenticated empty catalog.
- [ ] Unauthorized and nonexistent package routes are indistinguishable.
- [ ] Provider, module, version, documentation, governance, and copy flows work with keyboard and screen reader navigation.
- [ ] Loading, empty, expired, revoked, identity-outage, and API-outage states are verified.
- [ ] Markdown is sanitized and the browser loads no third-party runtime assets.
- [ ] Responsive and accessibility checks pass at supported breakpoints.
- [ ] Generated Registry favicon, PWA, and Open Graph assets render correctly.
