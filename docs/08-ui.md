# First-party Registry UI

## Product direction

The UI is a tracked React 19/Vite application. Its visual structure follows the compact utility patterns of the public Terraform Registry: a black product bar, full-width global search, blue artifact tabs, flat filter rails, dense result cards, and documentation-first package pages. It uses only the Registry name, original Registry emblem, authorized internal data, and local assets.

## Capabilities

- Providers, modules, and global authorized search.
- Truthful counts, facets, sorting, and cursor pagination.
- APM context selection for multi-membership and administrator users.
- Package/version navigation, documentation, governance, and one Artifactory install panel.
- Signed-in loading, no-entitlement, session-expired, identity-provider-down, revoked, empty, and API-error states.
- Authorized SSE refresh without page reload.
- Canonical plural routes with redirects from legacy singular deep links.

## Runtime boundary

The browser loads only non-secret runtime configuration: the same-origin API base path, display-only JFrog hostname, environment label, and support URL. OAuth, Graph, JFrog, and AWS credentials remain server-side.

The production image is a multi-stage Node/Nginx build with SPA fallback, immutable caching for hashed assets, no-cache HTML/runtime configuration, security headers, and an explicit health endpoint.

## Verification gate

Each release must pass unit/component tests, accessibility checks, and authenticated in-app-browser comparison at 1440, 1024, 768, and 390 pixels. The source reference and local screenshot must be evaluated in the same comparison image, and `web/design-qa.md` must end with `final result: passed`.
