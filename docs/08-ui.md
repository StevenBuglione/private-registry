# OpenTofu Registry UI Foundation

## Decision

Fork only the OpenTofu Registry UI frontend. Do not deploy the upstream backend, scraper, Cloudflare worker, public submission process, or public-scale storage/indexing architecture.

## Reused capabilities

- React/TypeScript/Vite application structure;
- registry-specific routes and page patterns;
- module/provider search and detail pages;
- version navigation;
- provider resources/data sources/functions/guides;
- module inputs/outputs/resources/examples/submodules;
- Markdown rendering and syntax highlighting;
- generated API types.

## Enterprise changes

Replace:

- upstream trademarks and branding;
- public submission/contribution links;
- public popularity/star assumptions;
- public moderation/license messaging that does not apply internally.

Add:

- runtime configuration loaded at container startup;
- organization-neutral branding and design tokens;
- owner/support/lifecycle/approval/security badges;
- generated JFrog source snippets;
- enterprise search facets;
- deprecated/revoked/replacement warnings;
- approvals, security, ownership, and restricted audit panels;
- safe authorization-aware error states.

## Patch boundary

Prefer:

```text
upstream component
  -> thin enterprise wrapper
  -> new enterprise components
```

Keep direct edits documented in `PATCHES.md`. Maintain an automated upstream intake branch, dependency/license diff, API contract tests, and visual regression suite.

## Runtime configuration

The same image runs in every environment. The Nginx entrypoint writes a non-secret JavaScript object containing:

- compatibility API base URL;
- enterprise API path;
- JFrog hostname used for display snippets;
- environment name;
- feature flags.

No token, client secret, AWS credential, or JFrog credential can appear in runtime browser configuration.

## Hosting

- multi-stage Node/Nginx image;
- non-root Nginx process on port 8080;
- SPA fallback;
- immutable caching for hashed assets;
- no-cache for HTML/runtime config;
- security headers and CSP;
- `/health/live` endpoint;
- ECS service across three AZs.

## Legal gate

Complete MPL-2.0 and third-party dependency/asset review. Preserve required notices and source obligations. Removing branding does not remove license obligations or grant trademark rights.
